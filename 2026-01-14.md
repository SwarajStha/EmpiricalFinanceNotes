- [slide 15] - decision trees are going to be different - 
	- majority voting = looking at what's the avg prediction from all the trees together => more robust than single decision tree; also won't be overfitted as much
- **bootstrap** - can create a lot of data from less data as changing one branch will lead to a very different tree
- **Re-substitution** -> should not be using, as uses all dataset and no data left for validation, as we already know the label for the target
- K-fold -> using similar split like simple training, but different iterations where one run includes different orientation of the dataset compared to another run
- Look-ahead bias even more prevalent/pronounced in machine learning than other research, as you are using hundreds of variables with thousands of parameters and you never know about all of them
- give-and-take => performance-complex to training time/intensity
